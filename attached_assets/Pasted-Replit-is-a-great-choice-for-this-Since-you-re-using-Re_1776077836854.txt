## 🛠️ Step 1: The Core Data Generator
Don't type the tables manually. Run this script in a Replit Python or JS file to generate your `data.json`. This ensures 100% accuracy for your users.

```javascript
const generateData = () => {
  const data = { tables: {}, squares: {}, cubes: {} };
  
  for (let i = 1; i <= 100; i++) {
    if (i <= 50) {
      data.tables[i] = Array.from({length: 10}, (_, t) => i * (t + 1));
      data.cubes[i] = Math.pow(i, 3);
    }
    data.squares[i] = Math.pow(i, 2);
  }
  return data;
};
```

## 💰 Step 2: Setting up AdMob (The Integration)
Since Replit is a web environment, you won't see "Real" AdMob ads during development (AdMob is for mobile apps). However, you should prepare your layout for them.

### How it will work:
1.  **Banner Placement:** Reserve a `div` at the bottom of your UI with a height of **50px**.
2.  **Interstitial Logic:** Create a "Counter" variable. After every **10 questions**, trigger an ad.
3.  **The Wrapper:** When you finish the code in Replit, you will move it to **Android Studio** and use a "WebView" with the `google_mobile_ads` SDK.

In bank exams, you get roughly **20-30 seconds** per question.
